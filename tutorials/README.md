## Contributing

When adding new tutorials:

1. **Choose the right category**: Create subdirectories for different types of content (troubleshooting, guides, examples)
2. **Use descriptive filenames**: Make it easy to find relevant content
3. **Include code examples**: Show both problematic and correct implementations
4. **Document the process**: Include step-by-step instructions and reasoning
5. **Link to related docs**: Reference relevant documentation and examples

## Categories

- **`troubleshooting/`** - Guides for fixing common issues and problems
- **`guides/`** - Step-by-step tutorials for specific tasks
- **`examples/`** - Additional code examples and demonstrations
- **`best-practices/`** - Recommended patterns and practices
